# 類似意見集約・件数集計プログラム 基本設計書

## 1. 文書情報
- 文書名：類似意見集約・件数集計プログラム 基本設計書
- 対象案件：AIを活用した中小企業向け勤怠管理関連コメント分析

## 2. システム概要
本システムは、CSV形式で入力された自由記述コメントを読み込み、AIによる類似度判定を用いて類似コメントをグルーピングし、件数集計結果をCSV形式で出力するバッチ型プログラムである。

また、名前やIPアドレス等の情報が入力に含まれる場合は、同一人物による重複投稿を判定し、重複を除いた人数集計も実施する。
本案件では誤判定を避けるため、同一人物判定は「氏名（フルネーム想定）+IPアドレスが両方一致した場合のみ」とする（厳しめ判定）。
加えて、グループに対してルールベースでラベルを自動付与し、出力CSVに含める。

## 3. システム構成
### 3.1 構成方針
- Windowsローカル環境で動作するPythonプログラムとする。
- GUIは持たず、ファイル入出力型のシンプルな構成とする。
- 処理単位ごとにモジュール分割し、保守性を確保する。

### 3.2 論理構成
1. 入力部
2. 前処理部
3. ベクトル化部
4. グルーピング部
5. 重複判定部
6. 集計部
7. 出力部

## 4. 処理方式
### 4.1 全体処理フロー
1. 入力CSVを読み込む。
2. 必須列存在チェックを行う。
3. コメントの前処理を行う。
4. コメントをベクトル化する。
5. 類似度に基づきグルーピングする。
6. グループごとに代表意見を決定する。
7. グループごとにラベルを自動生成する（ルールベース）。
7. 同一人物判定キーを生成する。
8. 重複排除後件数を集計する。
9. 集計結果CSVと明細CSVを出力する。

### 4.2 処理イメージ
- 入力：アンケートコメント一覧CSV
- 処理：類似分類、要約、件数集計、重複排除
- 出力：集計CSV、明細CSV

## 6. 画面設計
基本はバッチ型（コマンドライン起動）とするが、操作性向上のため Streamlit による簡易UIも提供する。

### 6.1 Streamlit 画面（簡易UI）
- CSVアップロード
- 実行ボタン
- 集計/明細の表示
- 集計CSV/明細CSVのダウンロード
- SQLiteに保存された過去実行の一覧表示

## 7. 機能設計
### 7.1 入力機能
- 指定フォルダまたは指定パスからCSVを読み込む。
- CSVの文字コードはUTF-8を基本とする。
- 必須列不足時はエラー終了とする。

### 7.2 前処理機能
- コメントの前後空白除去
- 改行の空白化
- 連続空白の正規化
- 空データ除外

### 7.3 類似判定機能
- コメント本文を埋め込みベクトルへ変換する。
- コメント間の類似度を計算する。
- 類似度閾値に基づきグループ化する。

### 7.4 代表意見生成機能
- グループ内で代表性の高いコメントを代表意見とする。
- 初期実装では、中心に最も近いコメントを代表文とする。
- 将来拡張としてLLM要約を追加可能とする。

### 7.5 重複判定機能
- 名前とIPアドレスから person_key を生成する（氏名+IPの両方がある場合のみ同一人物とみなす）。
- 同一グループ内における同一 person_key の2件目以降を重複と判定する。

### 7.5.1 ラベル自動生成機能（ルールベース）
- キーワード辞書によりグループ単位のラベルを自動付与する。
- 例）「使いやすい」「簡単」「分かりやすい」→ 操作性、「遅い」「重い」「固まる」→ 性能 等
- ラベル未該当の場合は「その他」とする。

### 7.6 集計機能
- グループごとのコメント件数を算出する。
- 重複除外後の人数を算出する。
- グループ単位の重複件数を算出する。

### 7.7 出力機能
- 集計結果CSVを生成する。
- 明細結果CSVを生成する。
- 出力先フォルダが存在しない場合は作成する。

### 7.8 DB保存・履歴管理機能（SQLite）
- 実行結果（集計/明細）を SQLite に保存する。
- 実行ごとに run_id を付与し、後から参照できる。
- 過去実行の検索ができる。
- 過去実行を削除できる（複数選択削除／全履歴全消し）。

## 8. 技術設計
### 8.1 採用技術
- Python 3.x
- pandas
- sentence-transformers
- scikit-learn
- numpy

### 8.2 AIエンジン方針
ローカルで再現しやすく、追加課金不要で試作品として提示しやすいことから、埋め込みモデルを利用した類似度判定方式を採用する。

#### 候補モデル
- intfloat/multilingual-e5-base
- paraphrase-multilingual-MiniLM-L12-v2

### 8.3 類似判定方式
- ベクトル化したコメントに対してコサイン類似度を算出する。
- 初期値として閾値0.80前後を採用し、設定で変更可能とする。

## 10. エラー処理方針
- 入力ファイルが存在しない場合はエラー終了する。
- 必須列が存在しない場合はエラー終了する。
- コメント有効件数が0件の場合はエラー終了する。
- モデル読み込み失敗時はエラーメッセージを出力する。
- 出力失敗時は異常終了とする。

## 11. ログ方針
- 開始ログ
- 入力件数ログ
- 前処理後件数ログ
- グループ数ログ
- 出力ファイルパスログ
- 異常終了時エラーログ

## 12. セキュリティ・個人情報配慮
- 名前、IPアドレス等は重複判定目的でのみ利用する。
- テスト作品ではサンプルデータを使用し、実在個人情報は含めない。
- 必要に応じてマスキング可能な設計とする。

## 13. 運用想定
- 利用者がCSVを input フォルダに格納する。
- プログラム実行後、output フォルダに結果CSVが出力される。
- 出力結果をExcelまたはPower BIで可視化する。
